<h2>COMENTARIO</h2>
Essa atividade mostra como criar um servidor em Python que entende as duas principais formas de conversa na internet: o GET e o POST. O código funciona como um atendente que identifica o que o usuário quer: se é apenas dar uma olhadinha em uma informação ou se é entregar dados novos para o sistema.

O método GET é usado para buscar informações, como quando você abre um site para ler uma notícia; ele é direto e rápido. Já o POST é usado para enviar dados, como quando você preenche um cadastro ou faz um login; ele é mais seguro porque esconde as informações dentro do "corpo" da mensagem, em vez de deixar tudo exposto no endereço do site. No seu teste, o servidor recebeu o nome e a escola que você enviou e confirmou que a entrega foi feita com sucesso.

Basicamente, o GET pede e o POST entrega. Ver as mensagens aparecendo no seu terminal confirma que essa troca de mensagens entre o cliente e o servidor está funcionando perfeitamente.
